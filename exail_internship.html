Leveraging Design Patterns for Python App development

Keywords: Python, Object Oriented Programming, Design Patterns, Agile Methodology, Client Needs Assesment


The challenge with this mission lies in the interconnectedness of all components within the Python application. For instance, if we add, remove, or modify the input values of any valorization step, all subsequent values in the following steps must update accordingly, and the shape of the 2D fiber representation must also adjust. To ensure the application remains well-organized and extensible for future features or actions, I implemented a design pattern: a combination of the builder and strategy patterns. This approach allowed for flexibility and better structure.
